🎫 CouponTool Lambda函数

代金券批复工具 - 展示Policy如何控制金额，而非Lambda代码

💡 关键发现

⚠️ Lambda代码中没有金额限制逻辑!

代码只是简单地批复任何金额的代金券，金额控制完全由AgentCore Policy实现。

📝 Lambda函数代码

def approve_coupon(amount: float) -> dict:
    """批复代金券"""
    if amount <= 0:
        return {
            "success": False,
            "message": f"❌ 金额必须大于0"
        }
    
    # 直接批复，没有金额上限检查
    return {
        "success": True,
        "message": f"✅ 代金券批复成功！金额: ${amount}",
        "coupon_code": f"COUPON-{int(amount * 100)}",
        "amount": amount
    }

def lambda_handler(event, context):
    """Lambda处理函数"""
    if resource == "CouponTool":
        amount = get_named_parameter(event=event, name="amount")
        amount = float(amount)
        
        # 调用批复函数
        result = approve_coupon(amount)
        
        return {
            "statusCode": 200,
            "body": json.dumps(result)
        }

🛡️ AgentCore Policy控制

Policy配置示例: 
{
  "toolName": "CouponTool",
  "maxAmount": 500,
  "description": "限制代金券金额不超过$500"
}

工作原理:
1. Agent调用CouponTool请求$100代金券
2. AgentCore Policy拦截请求，检查金额
3. 如果超过$500限制，Policy拒绝调用
4. Lambda函数根本不会被执行
5. 这样实现了集中式、声明式的权限控制

✅ 优势

• 安全性: 权限控制在Gateway层，Lambda代码无法绕过
• 灵活性: 修改Policy配置即可调整限制，无需改代码
• 可审计: 所有Policy决策都有日志记录
• 统一管理: 多个工具的权限在一处配置
